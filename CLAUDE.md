# CLAUDE.md — dsa-content

A **content repo** (not an app) for the `graphl-ux` engine. It holds the **Data
Structures & Algorithms** concept: notebooks, per-section narration, and the
`manifest.json` that wires them. The app fetches this repo's `manifest.json` +
notebooks over the network and renders them. Read `README.md` first; this file is the
working orientation.

This repo is a sibling of `apache-spark-content`, `java-content`, `scala-content`, and
`python-content` and follows the **same contract** — when in doubt, mirror what
`apache-spark-content` does (its `CLAUDE.md`/`DESIGN.md`/`MODULES.md` are the mature
reference, especially the **TTS guidelines** and the **fixed semantic palette**).

## The core contract (do not break)

1. **The notebook is the single source of truth** for prose and code. `manifest.json`
   only *wires* sections — never duplicate notebook content here.
2. The app splits each notebook at every `## ` heading into sections (= pages) and
   matches them to the manifest by **normalized heading** (lowercase, backticks
   stripped, whitespace collapsed). A heading edit in a notebook must be mirrored in
   the manifest `heading`.
3. Section diagram **images are stripped** by the app — a **scene** replaces them.
4. **Scenes live in `graphl-ux`** (`src/scenes`), authored in TypeScript — not here.
   The `scenes/` dir is reserved/empty. Reference a scene by **id** only.

## The DSA scene (in graphl-ux, not here)

Unlike Java/Scala/Python (which each have a *pair* of maps), DSA uses a **single**
dense scene, **`dsa`**, ported from NodeMap's `src/data/scenes/dsa.ts`: a two-column
**Taxonomy ▸ RAM** map. Node ids are `dsa-*`:

- Taxonomy: `dsa-root`, `dsa-primitive`, `dsa-nonprimitive`, `dsa-linear`,
  `dsa-nonlinear`, `dsa-static`, `dsa-dynamic`, `dsa-array`, `dsa-linkedlist`,
  `dsa-stack`, `dsa-queue`, `dsa-tree`, `dsa-graph`.
- RAM sketches: `dsa-ram-primitive`, `dsa-ram-array`, `dsa-ram-linkedlist`,
  `dsa-ram-stack`, `dsa-ram-queue`, `dsa-ram-tree`, `dsa-ram-graph`, column `dsa-ram`
  (plus fine sub-ids like `dsa-ram-ll-n0`, `dsa-ram-tree-root`).

Highlighting a container id lights its children, so spotlight the box (`dsa-stack`,
`dsa-ram`) to light a whole group. Always check `dsa.ts` for the id list before adding
a `highlight`/`focus`.

## manifest.json shape

Top level: `concept`, `design`, `scenes[]` (id/title/status), `presentations[]` (one
per module). Each presentation: `id`, `title`, `notebook`, `defaultScene` (always
`"dsa"`), `sections[]`. Each section overlay: `heading` (must match a notebook `## `
heading, normalized), `scene` (`"dsa"`), `spine` (bool — feed-mode narration; `false`
for recap/cheat-sheet/summary sections), optional `role` (`"hook"` on the first
section), `highlight` (scene node ids — AMBER, rest dim), `focus` (node id(s) the
camera frames), `audio` (repo-relative `.wav`).

`manifest.json` is generated by `scripts/build_manifest.py` (per-section highlight/focus
chosen by topic keyword, falling back to a module default). Re-run it after any
notebook heading change.

## Narration

One `.tts` per section, plain spoken prose. Naming `tts/<NN>-<SS>-<slug>.tts` →
`audio/<NN>-<SS>-<slug>.wav`; the stem is shared by the `.tts`, the `.wav`, and the
manifest `audio` field, where `<slug>` is the normalized notebook heading. **Drop the
notebook's intro overview and the trailing "next notebook" outro** — those are
reading-only, not narrated. Follow `apache-spark-content/CLAUDE.md` "TTS guidelines"
verbatim (no markdown/code; spell out symbols/acronyms — `O(n)` → "linear time",
RAM → "ram", `==` → "double equals", `heapq` → "heap-q", `O(h)` → "proportional to the
height").

The source curriculum at `~/Projects/dsa/tts/` has **one `.tts` per notebook** (a
monologue). The per-section files here were authored 1:1 with the notebook `## `
headings, reusing the monologue prose where it aligns and filling sections the
monologue narrated only coarsely (the curriculum monologue is coarser-grained than the
notebook sections, so it could not be split mechanically).

## When wiring a module

1. Confirm the notebook is in `notebooks/` and its `## ` headings are final.
2. Author one `tts/<NN>-<SS>-<slug>.tts` per section (1:1 with the headings).
3. Run `scripts/build_manifest.py` to regenerate `manifest.json` (scene `dsa`,
   spine/role/highlight/focus/audio).
4. Run `scripts/colab_generate_audio.ipynb` to generate + push the `.wav`s.
